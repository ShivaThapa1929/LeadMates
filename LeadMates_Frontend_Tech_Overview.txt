# 🧠 LeadMates — Frontend Technology Overview
> A plain-language guide explaining every technology used in the frontend and **exactly where** it shows up in the project.

---

## 📌 What Is the Frontend?

The **frontend** is everything the user **sees and clicks** in their browser — the pages, buttons, charts, forms, and animations. It lives in the `src/` folder and talks to the backend (API server) to fetch and save data.

---

## 🗂️ Project Folder Map

```
LeadMates/
├── src/
│   ├── main.jsx          ← App entry point (starts React)
```

## ⚙️ 1. Vite — Build Tool & Dev Server

| What it is | A super-fast tool that compiles and serves the app during development |
|---|---|
| **Used in** | [vite.config.js](file:///c:/Users/Admin/OneDrive/Desktop/LeadMates/LeadMates/vite.config.js), [package.json](file:///c:/Users/Admin/OneDrive/Desktop/LeadMates/LeadMates/package.json) scripts (`npm run dev`) |
| **Why it's here** | Replaces the old, slow Create React App. Starts the app instantly and reloads the browser when you save a file. |

**Key config in [vite.config.js](file:///c:/Users/Admin/OneDrive/Desktop/LeadMates/LeadMates/vite.config.js):**
- Runs the frontend at `http://localhost:5173`
- **Proxies** `/api` and `/uploads` requests to the backend at `http://localhost:5000` — so the browser never has CORS errors

## 🗺️ 3. React Router DOM v7 — Page Navigation

| What it is | A library that makes a single-page app behave like a multi-page website |
|---|---|
| **Used in** | [src/App.jsx](file:///c:/Users/Admin/OneDrive/Desktop/LeadMates/LeadMates/src/App.jsx) |
| **Why it's here** | Without it, changing the URL (`/login`, `/dashboard`, `/leads`) would reload the whole page. React Router handles navigation **instantly** without a page reload |

**Key routes defined in [App.jsx](file:///c:/Users/Admin/OneDrive/Desktop/LeadMates/LeadMates/src/App.jsx):**
| URL | Page shown |
|---|---|
| `/` | Home / Landing page |
| `/login` | LoginPage |
| `/signup` | SignupPage |
| `/dashboard` | Dashboard (protected — login required) |
| `/dashboard/leads` | LeadsPage |
| `/dashboard/analytics` | AnalyticsPage |
| `/dashboard/users` | UsersPage |
| `/dashboard/campaigns` | CampaignsPage |
| … and more | … |

**Special route guards in `components/`:**
- [RequireAuth.jsx](file:///c:/Users/Admin/OneDrive/Desktop/LeadMates/LeadMates/src/components/RequireAuth.jsx) — blocks pages if not logged in
- [RequireRole.jsx](file:///c:/Users/Admin/OneDrive/Desktop/LeadMates/LeadMates/src/components/RequireRole.jsx) — blocks pages based on user role (admin vs agent)
- [RequirePermission.jsx](file:///c:/Users/Admin/OneDrive/Desktop/LeadMates/LeadMates/src/components/RequirePermission.jsx) — blocks pages based on individual permissions
- [PublicRoute.jsx](file:///c:/Users/Admin/OneDrive/Desktop/LeadMates/LeadMates/src/components/PublicRoute.jsx) — redirects logged-in users away from Login/Signup

**Where you see it:**

**Also installed:** `tw-animate-css` — adds pre-made Tailwind animation classes (fade, slide, etc.)

## 🌐 7. Axios — HTTP Client (API Calls)

| What it is | A library for making HTTP requests (GET, POST, PUT, DELETE) to the backend |
|---|---|
| **Used in** | All files in `src/api/` |
| **Why it's here** | Used to talk to the backend server — fetch leads, save users, login, etc. |

**API service files (each one handles a different feature):**
| File | What it does |
|---|---|
| `authService.js` | Login, logout, signup, OTP, token refresh |
| `leadService.js` | Get, create, update, delete leads |
| `userService.js` | Manage users |
| `campaignService.js` | Campaign CRUD operations |
| `adminService.js` | Admin-only operations |
| `roleService.js` | Role management |
| `suspectService.js` | Suspect blends management |
| `trashService.js` | Trash/recycle bin |
| `customFieldApi.js` | Custom lead fields |

Axios is configured with **interceptors** in `authService.js` to automatically:
- Attach JWT tokens to every request
- Refresh expired tokens silently
- Redirect to login if the session is fully expired

## 🧩 8. Radix UI — Accessible UI Primitives

| What it is | A headless (no built-in styling) component library focused on accessibility |
|---|---|
| **Used in** | `src/components/ui/` (button, card, input, label, navigation-menu, separator) |
| **Why it's here** | Provides keyboard navigation, ARIA labels, and screen-reader support out of the box |

**Specific Radix packages used:**
- `@radix-ui/react-label` — accessible `<label>` for form fields
- `@radix-ui/react-navigation-menu` — the top navigation bar
- `@radix-ui/react-separator` — visual divider lines
- `@radix-ui/react-slot` — allows component composition (used in Button)

> These are styled with **shadcn/ui** patterns — meaning Radix provides the behavior and Tailwind provides the look.

## 🌓 10. React Context — Global State Management

| What it is | React's built-in way to share data across many components without passing it through every level |
|---|---|
| **Used in** | `src/context/` |
| **Why it's here** | Avoids "prop drilling" — passing the same data down 5 levels of components |

**Three context providers:**
| File | What it shares globally |
|---|---|
| `ThemeContext.jsx` | Dark mode / Light mode toggle — any component can read or change the theme |
| `NotificationContext.jsx` | Toast/notification system — any component can trigger a success or error message |
| `SearchContext.jsx` | Global search state — the search bar in the header shares results with pages |

## 🗺️ How a User Journey Maps to Technologies

```
User opens the app
  └─► Vite serves index.html
        └─► React mounts in the browser
              └─► React Router reads the URL and shows the correct page
                    ├─► Landing Page (Home.jsx + sections/)
                    │     └─► Tailwind CSS styling + Framer Motion animations
                    │
                    ├─► Login Page (LoginPage.jsx)
                    │     └─► Axios calls /api/auth/login → gets JWT token
                    │
                    └─► Dashboard (RequireAuth → Dashboard.jsx → Layout.jsx)
                          ├─► Tailwind CSS dark/light theme (ThemeContext)
                          ├─► Sidebar menu with Lucide icons
                          ├─► Recharts for analytics charts
                          ├─► Axios for all data fetching (leads, users, campaigns)
                          └─► Radix UI + shadcn components for forms/tables
```

---

> **In short:** React builds the pages, Vite compiles and serves them fast, React Router navigates between pages, Tailwind CSS makes them look great, Framer Motion adds animations, Axios talks to the backend, Recharts draws charts, and Radix UI makes forms and menus accessible.
